# Internationalization (i18n)

MDX entries support locale variants using a filename-based convention.

## File Naming

Add the locale code before the `.mdx` extension:

```
contents/blog/
  hello-world.mdx         # default locale
  hello-world.pt.mdx      # Portuguese
  hello-world.en-US.mdx   # English (US)
  hello-world.es.mdx      # Spanish
```

Supported patterns:
- Two-letter codes: `pt`, `es`, `fr`, `de`
- Region codes: `en-US`, `pt-BR`, `zh-CN`

## Querying by Locale

Use `.locale()` on the query builder:

```ts
// All Portuguese posts
const posts = queryCollection("blog")
  .locale("pt")
  .sort("date", "desc")
  .all();

// Single entry in a specific locale
const post = queryCollection("blog")
  .locale("es")
  .where({ slug: "hello-world" })
  .first();
```

Without `.locale()`, all locale variants are returned.

## ContentEntry

Locale entries have their locale stored in the `locale` field:

```ts
interface ContentEntry {
  collection: string;
  slug: string;       // "hello-world" (locale stripped)
  locale?: string;    // "pt", "en-US", etc.
  path: string;
  body?: string;
  data: Record<string, unknown>;
}
```

The slug is automatically cleaned — `hello-world.pt` becomes slug `hello-world` with locale `pt`.

## Studio UI

The MDX editor shows a locale switcher dropdown when multiple locale variants exist for an entry. Selecting a locale loads that variant for editing.

## API Reference

| Function | Description |
|----------|-------------|
| `parseLocaleFromFilename(filename)` | Extracts locale from `slug.locale.mdx` pattern |
| `stripLocaleFromSlug(slug, locale)` | Removes locale suffix from a slug |
| `queryBuilder.locale(code)` | Filters query results by locale |
